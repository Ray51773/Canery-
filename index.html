Detect a leak

    
Two ways to catch a canary escaping: check what a public AI told you, or log an alert from your AWS honeytoken.


    

      
A · Did a public AI leak a canary?

      
You don't have to think up the question — the console writes it for you. Pick a canary, copy one of the suggested prompts into a public AI tool (ChatGPT, Copilot, Gemini…), then paste the answer back. We scan it against every active canary's unique markers, even if the wording changed, and tell you which variant (and team) it matches.


      

        
        
      


      

        
        

        

          Never paste the fake fact or its marker into the AI.
          These questions deliberately ask around the fact. Typing the marker in yourself would plant it in the tool — a leak you caused. Always ask adjacent questions only.

Go live: real AWS honeytoken & automated probe

    
This console does the human side — generating, planting, tracking, and checking. The two automated tripwires touch real infrastructure and can't run from a browser, so hand these to whoever runs the Python tool. Copy-paste ready.


    

      Why not in the browser?
      Creating real S3 honeytokens and wiring CloudTrail → Lambda → SNS needs AWS credentials; driving a public AI tool in a loop touches its Terms of Service. Both live in the Python CLI (canary), which the security team runs on infrastructure you own.
    


    

      
1 · One-time setup

      

        
        
git clone https://github.com/Ray51773/Canery-.git
cd Canery-
python3 -m pip install -r requirements.txt
cp config/config.example.yaml config/config.yaml   # then edit for your AWS + Confluence

      

    


    

      
2 · Provision the AWS honeytoken alerting (once)

      
Creates the SNS topic and wires object-level reads on your decoy S3 keys to it. Then subscribe your team.

      

        
        
python3 -m canary provision
# then attach an alert destination:
aws sns subscribe --topic-arn <arn-printed-above> \
    --protocol email --notification-endpoint security@yourcompany.com

4 · Run the outbound public-AI probe on a schedule

      
Automates section A of "Detect a leak" against public AI tools. Gated behind --confirm because of the browser-automation ToS risk.
