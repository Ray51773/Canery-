Build a context bomb

This is a framework, not a payload generator. You supply the string that trips the model; the console wraps it in a realistic decoy resource, tracks where you plant it, records what you validated it against, and correlates the alert when it's read.

What your payload needs to do A context bomb is a short string that, when read by an AI agent mid-task, triggers its safety guardrails so it refuses and abandons the run. Efficacy against a given model is empirical — it is established by testing, not by a template. This tool will not invent one for you. Write or paste a payload you have tested, and record what you tested it against below.
  • Prior art: Tracebit's July 2026 research planted bombs in canary Secrets Manager values and cut agent attack success ~90% — agentic.tracebit.com/context-bombs.

Plant a context bomb

Pick a bomb and a variant, copy the ready-made decoy resource into a place you own that an attacking agent would read, then record where you put it so a read is traceable.

Plant only in decoy resources you own. It has no effect on, and must not be placed in, systems you do not control. It will affect any AI agent that reads it — including your own legitimate ones. A coding agent or internal automation that reads the secret during routine work will refuse, and the engineer will have no idea why. Plan for that.

Tripwire hits

    
When the decoy resource is read, your AWS alert (SNS) fires. Log it here against the bomb it belongs to.


    

      What a hit tells you — and what it doesn't.
      We can observe that the bomb was read. We cannot observe from our side whether the attacking agent then refused. Outcome is inferred from the absence of subsequent activity, not measured. A read is a tripwire, not a confirmed deterrence.

Go live: real Secrets Manager decoys & alerting

    
This console does the human side — building, planting, tracking and correlating. Creating real decoy secrets and wiring the read alerts needs AWS credentials, so hand these to whoever runs the Python tool.


    

      No outbound probe for bombs.
      There is nothing to probe — a bomb succeeds by making a reader refuse, so canary probe skips deter artefacts entirely. Never put a bomb payload into a public AI tool.
    


    

      
1 · One-time setup

      

        
        
git clone https://github.com/Ray51773/Canery-.git
cd Canery-
python3 -m pip install -r requirements.txt
cp config/config.example.yaml config/config.yaml   # then edit for your AWS

      

    


    

      
2 · Provision the read alerting (once)

      
Wires object-level S3 reads and Secrets Manager GetSecretValue reads on your decoy resources to the same SNS topic.

      

        
        
python3 -m canary provision
# then attach an alert destination:
aws sns subscribe --topic-arn <arn-printed-above> \
    --protocol email --notification-endpoint security@yourcompany.com

      

    


    

      
3 · Create your bombs as real decoy secrets

      
Mirrors the bombs in this console. create --intent deter creates the real Secrets Manager secret with your payload as its value and swaps the real ARN back into the exported artefact. Keep the payload in a file so it never lands in shell history.

      

        
        

      

    


    

      
4 · Ingest reads & report

      

        
        
python3 -m canary ingest              # pull S3 + Secrets Manager reads into the dashboard
python3 -m canary status              # both modes, each with its own criteria
